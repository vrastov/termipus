# Use Cases

## UC-1: AI агент работает над задачей в контексте MR

**Участники:** разработчик, AI агент (Kiro, Claude и др.)

**Сценарий:**
1. Разработчик открывает MR в GitLab
2. Нажимает кнопку "⌨ Terminal"
3. Сервер определяет репозиторий по URL страницы, проверяет наличие рабочей директории — если её нет, клонирует репозиторий. Создаёт worktree на ветке MR и запускает терминал в этом контексте.
4. Разработчик запускает AI агента — агент видит код ветки MR
5. Агент читает код, пишет изменения, запускает тесты прямо в терминале
6. CI запускается автоматически, результаты видны в MR
7. Разработчик наблюдает за работой агента и при необходимости вмешивается

**Ценность:** агент работает в контексте задачи, не переключаясь между инструментами.

## UC-2: Параллельная работа нескольких агентов

**Участники:** разработчик, несколько AI агентов

**Сценарий:**
1. Разработчик создаёт несколько Issue в GitLab
2. Для каждого Issue открывается отдельная вкладка браузера с терминалом
3. Сервер для каждой вкладки создаёт отдельный worktree на соответствующей ветке
4. Каждый агент работает в своём worktree независимо
5. Разработчик переключается между вкладками, наблюдает за прогрессом
6. Готовые MR проходят CI и ожидают ревью

**Ценность:** распараллеливание задач без конфликтов между агентами.

## UC-3: Ревью кода с AI ассистентом

**Участники:** ревьюер, AI агент

**Сценарий:**
1. Ревьюер открывает MR с изменениями
2. Запускает терминал с AI агентом прямо на странице MR
3. Просит агента объяснить изменения, найти потенциальные проблемы
4. Агент читает diff, запускает тесты, анализирует код
5. Ревьюер оставляет комментарии на основе анализа агента

**Ценность:** ускорение ревью, снижение когнитивной нагрузки.

## UC-4: Отладка в контексте Issue

**Участники:** разработчик

**Сценарий:**
1. Разработчик открывает Issue с описанием бага
2. Нажимает "⌨ Terminal" — сервер клонирует репозиторий если нужно, создаёт worktree на дефолтной ветке
3. Воспроизводит баг, запускает отладчик прямо в браузере
4. Фиксирует исправление, создаёт MR связанный с Issue

**Ценность:** весь контекст задачи (описание, код, терминал) в одном месте.

## UC-5: Запуск произвольных CLI инструментов

**Участники:** разработчик

**Сценарий:**
1. Разработчик открывает страницу проекта в GitLab/GitHub
2. Запускает терминал, подключённый к локальному серверу
3. Использует любые инструменты: `git`, `docker`, `kubectl`, скрипты деплоя
4. Результаты видны прямо в браузере без переключения в терминал

**Ценность:** браузер становится единой точкой входа для работы с проектом.
